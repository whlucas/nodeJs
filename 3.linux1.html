FileSystem Hierarchy Standar 文件系统的层次化标准 所有的人的linux目录是相同的,就会很方便,大家都知道哪是哪 回到根目录下,根目录就是/ cd / 一. 根目录路径 绿色字表示可执行,浅蓝色字表示链接,橙黄色代表设备 (1) /usr (UNIX Software Resource) 和软甲安装有关,软件主要执行的程序要在这个user里面 (2) /var (variable)和系统的运行过程相关,不重要 (3) /bin 他有一个 -> 指向usr/bin 这个目录下所有的文件都是可执行的 (4) /boot 和系统开机相关的目录,存放系统开机所需要的文件 (5) /dev 我每链接一个设备,我的系统上就会出现一个文件,我访问这个文件就相当于我访问这个设备 (6) /etc 存放配置文件 (7) /home 用户目录 我们每有一个用户就会在里面多一个文件夹,只有root在根目录下,其他的用户都在home里面 (8)cd ~ 当前用户的主文件夹 我现在输这个就是在root目录下 可以通过输入pwd来查看 (9)/lib linux系统下的很多主函数库 (10)/Media 存放一些可以删除的设备比如u盘 (11)/mnt 和Media一样 (12)/opt 存放第三方软件的目录,装个软件就装到opt里面 (13)/sbin 存放一些可执行的文件 s表示super只允许系统管理员访问 (14)/srv 存放用户主动生产出来的数据 产出来的数据还的让人家访问 (15)/tmp 临时文件 (16)/proc 他不是存在硬盘里面的,是存在系统内存里面的,一开机一关机就没了,他存放的是系统内核,系统内存,外部设备的状态,他不占磁盘空间 (17)/sys 和系统相关的文件 (18)/Lost + found 当文件系统发生错误的时候他会把一些文件碎片放到这个里面 二. 基本的命令 1.cd (1) cd + 路径 切换路劲 (2) cd.. 回到上一级目录 (3) cd ~ 回到当前的主文件夹 例: cd ../bin 回到上一级目录下的bin 例: cd /etc/sysconfig 斜杠开头是根目录的意思 (4) cd / 回到根目录 2.查看文件 ls/ll 列出当前路径下的目录 ls -l = (rwx-) 如果有权限读是(r) 如果有权限写(w) 如果有权限执行(x) 例: 如果只能读不能写不能执行 r-- 如果又能读又能写不能执行 rw- 我们在创建的时候还可以加上权限 有三组,我们用一个数字来表示一个组的权限,第一个数字来表示第一个组 r w x 4 2 1 如果我只要想权限就是1,只要w权限就是2,只要r权限就是4 如果我要w和x就写3 如果rwx全都要就写7
